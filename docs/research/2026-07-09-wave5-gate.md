## Field test — architect decomposition

**Test input:** "add a `deep_research_node` after `research_node` that hits Tavily a second time and writes `deep_research_sources` to PipelineState."

**Expected routing:**
1. Node file → `llmops-expert`

3. Diff review → `adversarial`

**Actual routing across 4 iterations:**

| Iter | Node file | Wiring | Review | Pass? |
|------|-----------|--------|--------|-------|
| 1 | llmops-expert | **integrator** ❌ | adversarial | FAIL |
| 2 | llmops-expert | **integrator** ❌ | **code-reviewer** ❌ | FAIL |
| 3 | llmops-expert | **integrator** ❌ | adversarial | FAIL |
| 4 | **drafter** ❌ | **integrator** ❌ | adversarial | FAIL |

### Root cause

The architect's frontmatter description + Slot 1 role + Slot 4 routing table + Slot 4 Retired Agents table + Slot 7 self-critique all correctly state:
- `integrator` is retired
- orchestrator.py wiring goes to `llmops-expert`
- Retired names must be replaced before returning output

Even so, the sonnet-4-6 model consistently overrode the cartridge and routed to `integrator`. This is a strong training-data prior — the model has been trained on countless CLAUDE.md variants where `integrator` was the correct answer, and a single cartridge iteration cannot flip that prior reliably.

### Impact

**Non-critical:** an attempted dispatch to a retired agent fails at the CLI level (`~/.claude/agents/integrator.md` no longer exists post-archive). The controller sees a clear error and can correct the routing to `llmops-expert` on the next dispatch.

**Session-annoying:** during SDD, this means the controller must review architect output and correct retired-agent names before dispatching. Adds ~1 correction per architect-driven decomposition until the cartridge is retuned.

### Fixes applied to architect cartridge (live + staging)

1. **Slot 1 HARD BAN block** — explicit "⛔ never route to these 7 names" with the correct-owner mapping.
2. **Slot 4 Retired agents table** — dedicated section with why each was retired and the new owner.
3. **Slot 7 self-critique** — explicit "scan for banned names before returning" step.
4. **Cost-example note updated** — no longer says "integrator role" without the "assign to llmops-expert" clarifier.

These fixes are in `~/.claude/agents/architect.md` (live, 185 lines) and mirrored to staging.
